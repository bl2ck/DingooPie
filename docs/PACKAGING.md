# Packaging Policy

The package is designed to ship the cleaned project source only. It
intentionally keeps local dependency caches and locally generated release
artifacts out of version control and out of source archives.

## Included

- Project source under `dingoo_pie/`
- CMake project files
- Build, bootstrap, and package scripts under `scripts/`
- Companion tools under `tools/`
- Project-owned PPSSPP patches under `patches/`
- Documentation under `docs/`

`downloads/`, `third_party/`, `deps_extract/`, `w64devkit/`, `build/`,
`release/`, and `dist/` are local workspaces. They are intentionally regenerated by
`scripts\bootstrap_windows.ps1`, `scripts\build_release.ps1`, and
`scripts\package_project.ps1` instead of being treated as project source.

## Excluded

- Game `.app` files, which belong to Dingoo Technology's package format
- Android `.apk` files
- Temporary CMake build trees
- Locally generated release binaries and DLLs
- Historical profile logs
- Debug screenshots and framebuffer dumps
- Dependency download caches and extracted dependency trees

## Rebuilding From the Package

Run:

```powershell
Set-ExecutionPolicy -Scope Process Bypass -Force
.\scripts\bootstrap_windows.ps1
.\scripts\build_release.ps1
```

`bootstrap_windows.ps1` creates `downloads/`, `third_party/`, `deps_extract/`,
and `w64devkit/` as needed. It reuses any matching local cache first. If an
archive is missing, it downloads the pinned dependency version.

## Updating Third-Party Code

PPSSPP source is downloaded into `third_party\ppsspp-master`. The Dingoo-specific changes must remain in project-owned patch files:

- `patches\ppsspp-irjit-dingoo.patch`

When refreshing PPSSPP, reapply the patch, rebuild, then run the smoke profile commands in `docs/DEBUGGING.md`.
Use legally obtained Dingoo A320 or compatible Gemei X760+ `.app` samples only
for local validation; never stage them into source-release archives.

## Verification

`scripts\package_project.ps1` writes archive-level `manifest.files.txt` and
`manifest.sha256`, creates the zip archive, extracts it into `dist\_verify`,
and validates every recorded hash. The script checks both the staged tree and
the verified archive for game files, logs, generated analysis files, and release
artifacts and fails if any forbidden file is found.
